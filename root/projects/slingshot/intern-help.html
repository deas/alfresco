Configuration

If you intend to run your tests against Sauce Labs, and assuming that you are using the deveng, please add the following configuration block to the end of your bash_profile. Obviously you will want to replace the username and access_key with those provided by Sauce Labs.
Note: Your bash_profile is usually found in your cygwin_home directory

# Sauce Labs configuration
export SAUCE_USERNAME={username}
export SAUCE_ACCESS_KEY={********-****-****-****-************}

Sauce Connect cannot tunnel localhost on anything but a few very standard ports. Port 8081 is not included in that list so it is necessary to provide a host lookup for 127.0.0.1 on your machine. Please add the following entry to your hosts file:

# Sauce Labs host lookup for 127.0.0.l
127.0.0.1	local.alfresco.test

Running tests with a local instance of Selenium

Before running tests against a local instance of Selenium make sure it is running. Please run the following commands from a new cygwin window with the correct project name:

$ workon project
$ cd projects/slingshot
$ node node_modules/intern/runner.js config=tests/intern_local.js

Running tests with a Virtual Machine

Before running tests against a local Virtual Machine you will need to bring the machine up. Please refer to the Vagrant help file if you do not know how to do that. If you are happy that the VM is running, please run the following commands from a new cygwin window with the correct project name:

$ workon project
$ cd projects/slingshot
$ node node_modules/intern/runner.js config=tests/intern.js

Running tests with Sauce Labs

Before running tests against Sauce Labs you will need to adhere to the configuration details shown above. If you are happy that the configuration is complete, please run the following commands from a new cygwin window with the correct project name:

$ workon project
$ cd projects/slingshot
$ node node_modules/intern/runner.js config=tests/intern_sl.js
